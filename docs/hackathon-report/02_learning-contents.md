## 🎯 コンテンツ設計の方針

### 1. **段階的な学習曲線**

```
難易度 ⭐     → ⭐⭐    → ⭐⭐⭐
基礎セットアップ  応用テクニック  高度な活用
```

- **⭐⭐（中級）**: 議事録生成、Plan mode、通知設定

### 2. **実践的なユースケース**

すべてのコンテンツが、実際の業務で使えるテクニックに基づいている。

| コンテンツ | 実務での活用例 |

## 📝 各コンテンツの詳細

### 01. VSCodeでClaude Codeを使えるようにしよう 🔧

**難易度**: ⭐ | **所要時間**: 10分

#### 学習目標
- Claude Codeとは何かを理解する
- VSCodeにClaude Code拡張機能をインストールする
- APIキーまたはAWS Bedrockを設定する
- 初めてのClaude Codeコマンドを実行する

#### コンテンツ構成
1. **イントロダクション**: Claude Codeの概要とメリット
2. **前提条件の確認**: VSCodeインストール済みか確認
3. **セットアップ方法の選択**:
   - 📱 **直接API接続**（個人開発者向け）
   - ☁️ **AWS Bedrock経由**（企業・チーム向け）
4. **ステップバイステップ手順**: 各方法の詳細な手順
5. **動作確認**: Hello Worldで動作テスト
6. **便利な機能の紹介**: Chat、Edit、Fix、Explainコマンド

#### 技術的な特徴
- **2つの接続方法を並列提示**: タブUIで切り替え可能
- **AWS Bedrock対応**: IAM設定、AWS CLI設定まで完全ガイド
- **トラブルシューティング**: よくあるエラーと対処法

#### 関連ファイル
- `pages/01_🔧_VSCode設定.py`
- `docs/specs/SPEC-vscode-setup.md`

## 🔄 コンテンツ追加の仕組み

### YAMLベースの管理

`contents.yaml`にエントリを追加するだけで、新しいコンテンツがアプリに反映される。

```yaml
- id: "08"
  title: "新しいコンテンツ"
  description: "説明文"
  duration: "20分"
  difficulty: 2
  page: "08_新コンテンツ"
  icon: "🎉"
```

### 追加手順

1. **SPEC作成**: `docs/specs/SPEC-newcontent.md`
2. **ページ実装**: `pages/08_🎉_新コンテンツ.py`
3. **YAML追加**: `contents.yaml`にエントリ追加
4. **デプロイ**: Gitにpushすれば自動反映

詳細は `docs/guides/add-content.md` を参照。
