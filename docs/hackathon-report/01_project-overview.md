## 🎯 プロジェクトの目的

### なぜこのアプリを作ったのか

**背景:**
- 社内メンバー（特に非エンジニア）がAI活用を学ぶハードルが高い
- ブログ記事を読むだけでは、実際の活用イメージが湧かない
- 「Claude Codeを使ってみたいけど、何から始めればいいかわからない」という声

**目的:**
1. **体験型学習**：読むだけでなく、実際に手を動かして学べる
2. **段階的習得**：簡単なコンテンツから始めて、徐々にレベルアップ
3. **実践的スキル**：すぐに業務で使えるテクニックを習得
4. **メタ学習**：YAMLファイルの構造を学び、自分でコンテンツを追加できる

### ターゲットユーザー

1. **非エンジニア社員**
   - 総務、営業、マーケティングなど
   - AIツールに興味はあるが、技術的な知識がない
   - 実践的なスキルを短時間で習得したい

2. **エンジニア（初級〜中級）**
   - Claude Codeの基本的な使い方を学びたい
   - 仕様駆動開発のプラクティスを知りたい
   - 効率的な開発フローを構築したい

3. **社内AI活用推進者**
   - 他部署へのAI活用を支援したい
   - 実践的な教材が欲しい
   - 社内研修のコンテンツを探している

## 💡 コンセプト

### 「みのるん流」AI活用の再現

みのるん氏（@minorun365）のブログ記事「Claude Codeで月末業務を5分で終わらせる話」を参考に、以下の考え方を実装：

#### 1. **100%自動化より、一番つらい部分だけ自動化**
- 完全自動化は難しい（権限、セキュリティ、API制限）
- 手動操作を残しつつ、時短を実現
- 例: 経費精算の勘定科目分類だけAIに任せる

#### 2. **プチ仕様駆動開発**
- **PLAN.md**: 音声入力で思考をダンプ
- **SPEC.md**: Claudeと壁打ちして認識齟齬をなくす
- **TODO.md**: タスク管理
- **KNOWLEDGE.md**: ハマったことを記録

→ このプロジェクト自体が、プチ仕様駆動開発で作られている！

#### 3. **音声入力の重要性**
- タイピングでは出せない情報量
- フィラー混じりでOK
- 10分で数千文字

→ 将来的にWeb Speech API統合を予定

#### 4. **モノレポでの雑務管理**
- 小さな作業を1つのリポジトリで管理
- Claudeがコンテキストを保持
- README.mdで手順・ハマりポイントを記録

## 🛠️ 技術スタック

### Frontend
- **Streamlit** (v1.42.1)
  - Pythonで手軽にWebアプリを構築
  - インタラクティブなUI要素が豊富
  - データ可視化に強い

### Backend
- **Python** (3.12.8)
  - データ処理、YAML読み込み
  - Streamlitとのシームレスな統合

### AI/LLM
- **Claude Code**
  - コード生成、リファクタリング
  - 仕様書作成支援
  - ドキュメント自動生成

### インフラ・デプロイ
- **Streamlit Community Cloud**
  - 無料でWebアプリをホスティング
  - GitHubと自動連携
  - カスタムドメイン対応

### 設定管理
- **YAML**
  - コンテンツメタデータの管理
  - 人間が読み書きしやすい
  - プログラム的にも扱いやすい

### バージョン管理
- **Git/GitHub**
  - トランクベース開発（mainブランチ直接）
  - コミットメッセージにClaude Code署名

### UI/UXの特徴

1. **グラデーション背景**
   - ヘッダーに視覚的な魅力
   - プロフェッショナルな印象

2. **カード型レイアウト**
   - 学習コンテンツを一覧で見やすく
   - 難易度・所要時間を明示

3. **サイドバー非表示**
   - すっきりとしたUI
   - コンテンツに集中できる

4. **TOPボタン常設**
   - どのページからでもトップに戻れる
   - ナビゲーションの迷子を防ぐ
