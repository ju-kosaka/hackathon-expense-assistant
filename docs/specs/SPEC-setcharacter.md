### 2. `.claude/CLAUDE.md`の役割

**図解で説明**:
```
プロジェクトルート/
├── .claude/
│   └── CLAUDE.md  ← ここにキャラ設定を書く！
├── .gitignore      ← .claude/を除外設定
├── app.py
└── pages/
```

**重要ポイント**:
- `.claude/`フォルダはClaude Code専用のフォルダ
- `CLAUDE.md`にMarkdown形式でキャラクター情報を記述
- Claude Code起動時に自動で読み込まれる
- `.gitignore`に`.claude/`を追加すると、Gitで管理されない（個人設定として保持）

### 5. Gitにpushされない仕組みの解説

**なぜ`.gitignore`に追加するの？**
- キャラクター設定は**個人的な好み**
- チームメンバーと共有する必要がない
- プライバシーを守るため（例: 個人的なメモが含まれている場合）

**仕組みの図解**:
```
あなたのPC              GitHub（リモート）
├── .claude/           
│   └── CLAUDE.md  →  pushされない（.gitignoreで除外）
├── app.py         →  pushされる
└── pages/         →  pushされる
```

**メタ学習ポイント**:
- `.gitignore`の役割を体験的に理解
- ローカル専用の設定ファイルの管理方法を学ぶ
